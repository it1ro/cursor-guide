# Принципы работы с Cursor

## Cursor — это инженер, а не IDE

Относись к Cursor как ко второму разработчику:
- Объясняй архитектуру
- Задавай роль
- Ставь задачу
- Указывай ограничения
- Требуй качество

Cursor следует твоей инженерной рамке, а не угадывает.

## Контекст важнее задачи

Без контекста Cursor придумывает своё.

С контекстом Cursor:
- следует архитектуре
- не ломает стилистику
- не вводит новые абстракции
- не нарушает принципы

Контекст = архитектура → цели → ограничения → стиль → принципы.

## Cursor работает итеративно

Одна большая задача → плохой результат.

Много маленьких итераций → идеальный результат.

Cursor любит:
- уточнения
- поэтапные задачи
- пошаговые улучшения
- "сначала предложи варианты"

## Cursor нужно ограничивать

Без рамок Cursor:
- создаёт лишние слои
- переусложняет
- меняет API
- нарушает принципы

Всегда указывай:
- что не менять
- что не добавлять
- какие принципы соблюдать

## Cursor — для архитектурных обсуждений

Cursor умеет:
- сравнивать подходы
- предлагать варианты
- объяснять компромиссы
- помогать формировать roadmap

Но не принимает решения за тебя — это генератор вариантов, а не архитектор.

## Cursor — мощный исполнитель при точной задаче

При чёткой задаче Cursor:
- пишет код
- рефакторит
- мигрирует
- создаёт тесты
- обновляет документацию
- правит несколько файлов сразу

Главное — точность формулировки.

## Задавай роль

Примеры:
- "Ты senior TypeScript engineer"
- "Ты архитектор проекта"
- "Ты эксперт по физике полёта"
- "Ты редактор документации"

Роль = стиль + глубина + ограничения.

## Cursor не думает за тебя

Ты — архитектор. Cursor — исполнитель и генератор вариантов.

Ты формируешь:
- требования
- ограничения
- принципы
- критерии качества

Cursor — реализует.
