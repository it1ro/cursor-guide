# 10. Личные практики

## 10.1. Практика: Cursor как первый собеседник, а не первый исполнитель

Ты всегда начинаешь с обсуждения:

- архитектуры,
- вариантов,
- компромиссов,
- API,
- структуры данных.

Cursor помогает тебе думать, но **не принимает решения за тебя**.

Ты — архитектор.  
Cursor — генератор вариантов.

---

## 10.2. Практика: Итеративный цикл разработки (твоя фирменная схема)

Ты работаешь по схеме:

1. **Требования**
2. **Ограничения и компромиссы**
3. **Выбор инструментов**
4. **Roadmap**
5. **Детализация этапов**
6. **Реализация**
7. **Тесты**
8. **Документация**

Cursor идеально вписывается в каждый шаг:

- помогает формулировать требования,
- уточняет ограничения,
- предлагает варианты инструментов,
- строит roadmap,
- помогает декомпозировать,
- пишет код,
- пишет тесты,
- пишет документацию.

---

## 10.3. Практика: Минимальный контекст → максимальная ясность

Ты всегда даёшь Cursor:

- 3–7 строк контекста,
- 3–5 ограничений,
- 1 чёткую задачу,
- 1 формат ответа.

Это даёт **предсказуемый, чистый результат**.

---

## 10.4. Практика: Cursor + Obsidian

Ты используешь Obsidian ежедневно.  
Cursor идеально дополняет его:

- генерируешь выжимки → вставляешь в Obsidian,
- создаёшь архитектурные заметки,
- формируешь roadmap,
- сохраняешь usage examples,
- ведёшь журнал решений.

Cursor → Obsidian → твой мозг → код.

---

## 10.5. Практика: Cursor + Anki

Ты используешь Anki для закрепления знаний.  
Cursor может:

- генерировать карточки по архитектуре,
- создавать карточки по физике,
- формировать карточки по API,
- помогать закреплять паттерны.

Это превращает твой стек знаний в **постоянно растущую систему**.

---

## 10.6. Практика: Cursor + твой dev-стек (EndeavourOS, tmux, zsh, lazy.nvim)

Cursor идеально вписывается в твой рабочий стиль:

- tmux → быстрые сессии разработки
- zsh → быстрые команды
- lazy.nvim → лёгкая навигация
- EndeavourOS → стабильная среда
- Cursor → интеллектуальный помощник

Ты используешь Cursor как:

- генератор кода,
- рефакторинг-инструмент,
- архитектурный анализатор,
- документационный движок.

---

## 10.7. Практика: Cursor как инструмент "чистоты"

Ты используешь Cursor для:

- поиска несогласованностей,
- проверки архитектуры,
- поиска скрытого состояния,
- поиска nondeterminism,
- проверки naming conventions.

Cursor становится **стражем чистоты** твоего кода.

---

## 10.8. Практика: Cursor как ускоритель R&D

Ты используешь Cursor для:

- быстрых прототипов,
- mock-модулей,
- проверки гипотез,
- сравнения моделей,
- анализа формул.

Это ускоряет разработку flight-engine и Airblox.

---

## 10.9. Практика: Cursor как инструмент документации

Ты используешь Cursor для:

- TS-only документации,
- генерации usage examples,
- создания структурированных описаний,
- обновления документации при изменениях.

Документация становится **живой**.

---

## 10.10. Практика: Cursor как часть твоего мышления

Ты используешь Cursor как:

- зеркало для идей,
- инструмент уточнения,
- способ проверить гипотезу,
- помощника в принятии решений.

Cursor усиливает твой стиль:

- минимализм,
- ясность,
- модульность,
- предсказуемость,
- чистота архитектуры.
