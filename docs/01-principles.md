# 1. Принципы работы с Cursor AI

## 1.1. Cursor — это не IDE, а инженер

Cursor работает лучше всего, когда ты относишься к нему не как к инструменту автодополнения, а как к **второму разработчику**, которому можно:

- объяснить архитектуру,
    
- дать роль,
    
- поставить задачу,
    
- обозначить ограничения,
    
- потребовать качество.
    

Cursor не угадывает — он **следует твоей инженерной рамке**.

## 1.2. Контекст важнее задачи

Если дать задачу без контекста, Cursor будет “умничать” и придумывать своё.

Если дать контекст, он:

- следует архитектуре,
    
- не ломает стилистику,
    
- не вводит новые абстракции,
    
- не нарушает принципы проекта.
    

Контекст = архитектура → цели → ограничения → стиль → принципы.

## 1.3. Cursor лучше всего работает итеративно

Одна большая задача → плохой результат. Много маленьких итераций → идеальный результат.

Cursor любит:

- уточнения,
    
- поэтапные задачи,
    
- пошаговые улучшения,
    
- “сначала предложи варианты”.
    

## 1.4. Cursor нужно ограничивать

Если не поставить рамки, он:

- создаёт лишние слои,
    
- переусложняет,
    
- меняет API,
    
- нарушает принципы проекта.
    

Поэтому всегда указывай:

- что **не** менять,
    
- что **не** добавлять,
    
- какие принципы соблюдать.
    

## 1.5. Cursor — идеален для архитектурных обсуждений

Он отлично:

- сравнивает подходы,
    
- предлагает варианты,
    
- объясняет компромиссы,
    
- помогает формировать roadmap.
    

Но он не должен принимать решения за тебя — он **генератор вариантов**, а не архитектор.

## 1.6. Cursor — мощный исполнитель, если задача точная

Когда задача чёткая, Cursor:

- пишет код,
    
- рефакторит,
    
- мигрирует,
    
- создаёт тесты,
    
- обновляет документацию,
    
- правит несколько файлов сразу.
    

Главное — **точность формулировки**.

## 1.7. Cursor работает лучше всего, когда ты задаёшь роль

Примеры ролей:

- “Ты senior TypeScript engineer”
    
- “Ты архитектор проекта”
    
- “Ты эксперт по физике полёта”
    
- “Ты редактор документации”
    

Роль = стиль + глубина + ограничения.

## 1.8. Cursor не должен думать за тебя

Ты — архитектор. Cursor — исполнитель и генератор вариантов.

Ты формируешь:

- требования,
    
- ограничения,
    
- принципы,
    
- критерии качества.
    

Cursor — реализует.
