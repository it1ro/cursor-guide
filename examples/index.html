    Примеры из проектов¶

    telnik-flight-engine¶

    Проектирование модуля Drag¶

    Ситуация: нужен минимальный модуль аэродинамического сопротивления.

    Проблема: Cursor склонен добавлять классы, новые типы, "умные" абстракции.

    Решение: 

    Pure function only. No classes. No new abstractions. Deterministic. TS-only.
Propose 3 variants. Compare. Recommend minimal.

    Результат: Cursor предложил три варианта. Выбран самый простой — чистая функция computeDrag. Стала эталоном для всех будущих модулей.

    Проверка формулы лифта¶

    Ситуация: нужно убедиться, что формула лифта корректна для low-speed arcade physics.

    Решение: 

    Check this formula for correctness, edge cases, and numerical stability.

    Результат: Cursor нашёл: - проблему при нулевой скорости - необходимость clamp для отрицательных значений - потенциальную нестабильность при больших скоростях

    Это улучшило реализм и стабильность симуляции.

    Multi-file refactor¶

    Ситуация: нужно переименовать функцию во всём проекте.

    Решение: 

    Show list of files first. Show plan. Wait for confirmation.

    Результат: Cursor изменил только нужные файлы, не тронув архитектуру.

    Генерация usage examples¶

    Ситуация: нужно создать примеры использования для документации.

    Решение: минимальный контекст и строгий формат.

    Результат: Cursor создал чистые, понятные примеры, которые стали частью TS-only документации.

    Airblox¶

    Прототипирование физики самолёта¶

    Ситуация: нужно быстро проверить несколько моделей поведения самолёта.

    Решение: 

    Create a simplified prototype. Pure function. Minimal inputs. Deterministic.

    Результат: Cursor создал mock-модуль, который позволил быстро протестировать идею без полноценной реализации.

    Проектирование структуры состояния¶

    Ситуация: нужно определить структуру состояния самолёта.

    Решение: 

    Design a minimal, explicit, extensible state structure.

    Результат: Cursor предложил структуру, которую затем уточнили и сделали частью движка.

    Обсуждение компромиссов¶

    Ситуация: нужно выбрать модель поведения самолёта для Airblox.

    Решение: попросили Cursor сравнить: - реалистичную модель - упрощённую модель - аркадную модель

    Результат: Cursor дал таблицу trade-offs, которая помогла принять решение.

    Telnik Platform¶

    Документация TS-only¶

    Ситуация: нужна документация: - минималистичная - точная - без воды - пригодная для AI-агентов

    Решение: 

    TS-only documentation. No narrative. Include formula, parameters, usage example.

    Результат: Cursor создал документацию, которая стала источником правды для проекта.

    Аудит архитектуры¶

    Ситуация: нужно проверить согласованность модулей.

    Решение: 

    Check for architectural inconsistencies, naming issues, hidden state, nondeterminism.

    Результат: Cursor нашёл несколько мелких несоответствий, которые исправили.

    Roadmap для миграции¶

    Ситуация: нужно спланировать миграцию документации и модулей.

    Решение: 

    Create a roadmap with steps, dependencies, risks, testing strategy.

    Результат: Cursor создал чёткий план, который использовали как основу.
