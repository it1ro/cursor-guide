    Практики работы с Cursor¶

    Cursor как первый собеседник, а не первый исполнитель¶

    Всегда начинай с обсуждения: - архитектуры - вариантов - компромиссов - API - структуры данных

    Cursor помогает думать, но не принимает решения за тебя.

    Ты — архитектор. Cursor — генератор вариантов.

    Итеративный цикл разработки¶

    1. Требования
    2. Ограничения и компромиссы
    3. Выбор инструментов
    4. Roadmap
    5. Детализация этапов
    6. Реализация
    7. Тесты
    8. Документация

    Cursor вписывается в каждый шаг: - помогает формулировать требования - уточняет ограничения - предлагает варианты инструментов - строит roadmap - помогает декомпозировать - пишет код - пишет тесты - пишет документацию

    Минимальный контекст → максимальная ясность¶

    Всегда давай Cursor: - 3–7 строк контекста - 3–5 ограничений - 1 чёткую задачу - 1 формат ответа

    Это даёт предсказуемый, чистый результат.

    Cursor + Obsidian¶

    Используй Obsidian ежедневно. Cursor дополняет его: - генерируешь выжимки → вставляешь в Obsidian - создаёшь архитектурные заметки - формируешь roadmap - сохраняешь usage examples - ведёшь журнал решений

    Cursor → Obsidian → твой мозг → код.

    Cursor + Anki¶

    Используй Anki для закрепления знаний. Cursor может: - генерировать карточки по архитектуре - создавать карточки по физике - формировать карточки по API - помогать закреплять паттерны

    Это превращает твой стек знаний в постоянно растущую систему.

    Cursor + dev-стек¶

    Cursor вписывается в рабочий стиль: - tmux → быстрые сессии разработки - zsh → быстрые команды - lazy.nvim → лёгкая навигация - EndeavourOS → стабильная среда - Cursor → интеллектуальный помощник

    Используй Cursor как: - генератор кода - рефакторинг-инструмент - архитектурный анализатор - документационный движок

    Cursor как инструмент "чистоты"¶

    Используй Cursor для: - поиска несогласованностей - проверки архитектуры - поиска скрытого состояния - поиска nondeterminism - проверки naming conventions

    Cursor становится стражем чистоты кода.

    Cursor как ускоритель R&D¶

    Используй Cursor для: - быстрых прототипов - mock-модулей - проверки гипотез - сравнения моделей - анализа формул

    Это ускоряет разработку flight-engine и Airblox.

    Cursor как инструмент документации¶

    Используй Cursor для: - TS-only документации - генерации usage examples - создания структурированных описаний - обновления документации при изменениях

    Документация становится живой.

    Cursor как часть мышления¶

    Используй Cursor как: - зеркало для идей - инструмент уточнения - способ проверить гипотезу - помощника в принятии решений

    Cursor усиливает твой стиль: - минимализм - ясность - модульность - предсказуемость - чистота архитектуры
